Agentic self-healing: yes, this works today. Pattern: agent hits a task with no skill for it → inserts into agent_notifications "No skill exists for [X], recommend building" → Chief of Staff surfaces it in morning report → You say yes → skill-creator skill (already built into Cowork) generates the new skill folder → it's live for the next run. A website publisher agent doesn't need its own Core Term knowledge. It calls the core-term-audit skill. Skills compose -- you don't duplicate knowledge across skills, you reference it. The _shared/ folder is the foundation everything sits on. That's where the blade mandate lives. That's where the feedback loop lives. Each specialist skill is thin because _shared/ is deep.

For Bodhi 360 scale: same architecture, different _shared/ per project. Mirror's shared knowledge doesn't leak into Harmonic. Harmonic's day-job context doesn't contaminate Mirror's anonymity rules. Isolated by design. One Claude. Right skills loaded. Right context only.
